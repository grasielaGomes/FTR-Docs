### Uso Seguro de Inteligência Artificial: Boas Práticas e Técnicas de Proteção

#### 1. Quando é Seguro Utilizar IA

- Utilizar apenas dados completamente anonimizados ou sintéticos.
- Garantir que dados utilizados em práticas ou testes estejam anonimizados.
- Consultas sem informações sensíveis ou confidenciais.
- Realizar prototipagem e testes sem conexão com dados reais.

#### 2. Cuidados Práticos

- Certificar-se de que os dados utilizados estão anonimizados ou são sintéticos.
- No curso, apenas dados sintéticos são utilizados para evitar vazamentos.

#### 3. Técnicas de Proteção de Dados

- **Tokenização:** Substituir dados reais por tokens para proteger informações sensíveis.
- **Mascaramento de Dados:** Manter os dados presentes, mas não visíveis (ex: mascarar CPF), útil para compartilhamento entre times sem expor informações.
- **Ambientes de Sandbox Internos:** Utilizar IA licenciada e implementada internamente para garantir que dados não sejam vazados ou utilizados pela provedora da IA.
- **Consentimento:** Obter autorização do usuário para uso dos dados, quando necessário, incluindo cláusulas nos termos de uso do software ou aplicativo.

#### 4. Considerações sobre Consentimento

- Solicitar consentimento apenas quando apropriado e necessário.
- Nem sempre é adequado pedir autorização para uso de dados sensíveis em IA externa.
- Consentimento pode ser obtido por meio de termos de uso assinados pelo usuário ao acessar o software.

#### 5. Recomendações Finais

- Avaliar cuidadosamente o tipo de dado e a situação antes de utilizar IA externa.
- Utilizar técnicas de proteção e ambientes controlados para garantir compliance e segurança.
- Ter parcimônia ao solicitar consentimento, pois o cliente pode recusar o uso dos dados.
