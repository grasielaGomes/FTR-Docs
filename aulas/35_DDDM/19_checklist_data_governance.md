### Checklist de Data Governance: Pontos Essenciais para Uso Seguro de Dados

#### 1. Confidencialidade dos Dados

- Verifique se os dados são confidenciais.
- Avalie a necessidade de proteção extra para dados de stakeholders ou clientes.

#### 2. Anonimização dos Dados

- Confirme se os dados foram anonimizados (tokenização, mascaramento, etc).
- Caso não estejam anonimizados, defina como será feita a anonimização.
- Escolha a técnica mais adequada para o tipo de dado e análise.

#### 3. Risco de Vazamento

- Avalie se existe risco de vazamento mesmo após anonimização.
- Considere se o uso em IA ou outras ferramentas pode gerar exposição indesejada.

#### 4. Políticas Internas e Legislação

- Verifique se o uso dos dados viola políticas internas da empresa.
- Avalie se há infração de leis de proteção de dados (LGPD, GDPR, HIPAA, etc).
- Consulte legislações vigentes de acordo com o país de atuação.

#### 5. Uso de Dados Fictícios ou Sintéticos

- Considere criar dados fictícios para testes, caso não seja possível usar dados reais.
- Certifique-se de que os dados sintéticos sejam representativos, mas não distorçam a análise.
- Evite alterações bruscas que possam gerar conclusões erradas.

#### 6. Cuidados ao Modificar Dados

- Ao criar dados sintéticos, mantenha parâmetros próximos da realidade.
- Evite maquiar dados de forma que comprometa a análise ou gere interpretações incorretas.

#### 7. Responsabilidade no Uso dos Dados

- Trate os dados como ativos valiosos e sensíveis.
- Reflita sobre a responsabilidade de manipular e proteger informações.
- Siga processos seguros antes de utilizar IA ou compartilhar dados.

#### 8. Importância do Dado

- Valorize o dado desde o início do processo.
- Lembre-se de que a etapa mais importante é a definição e proteção do dado, não apenas o uso final em IA ou análises.

#### 9. Dica Prática

- Mantenha o checklist sempre à mão (post-it, mural, etc) para consulta rápida ao receber ou trabalhar com novos dados.
