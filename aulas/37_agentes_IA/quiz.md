## Quiz Avaliativo – Agentes de IA

---

**1. O que define um agente de IA dentro do contexto apresentado no módulo?**

- Um modelo de linguagem que apenas responde perguntas
- Um banco de dados treinado com respostas predefinidas
- Um programa fixo sem capacidade de adaptação
- **Um sistema que age de forma autônoma com base em percepções e objetivos**

---

**2. Qual é o principal propósito do paradigma ReAct (Reasoning + Acting)?**

- Aumentar a velocidade de respostas do modelo
- Gerar textos mais criativos com menos contexto
- Melhorar a tradução de idiomas em modelos de IA
- **Permitir que o agente raciocine e execute ações de forma integrada**

---

**3. No contexto de agentes, o que são 'tools' ou ferramentas?**

- Arquivos de configuração do modelo de linguagem
- Bibliotecas de código-fonte da IA
- Extensões visuais para o ambiente de desenvolvimento
- **Funções externas que o agente pode acionar para realizar tarefas específicas**

---

**4. Qual é o papel do Model Context Protocol (MCP) em agentes de IA?**

- Executar o raciocínio lógico do agente
- Armazenar as respostas em cache para reutilização
- Gerar prompts automaticamente para o modelo
- **Permitir a comunicação segura e padronizada entre o modelo e sistemas externos**

---

**5. O que caracteriza um sistema multiagente?**

- A execução de várias threads por um mesmo modelo
- A otimização de prompts em sequência
- A substituição de agentes humanos por IA
- **A cooperação entre múltiplos agentes com objetivos complementares**

---

**6. Por que o LangGraph é importante na construção de agentes de IA?**

- Porque substitui completamente o MCP
- Porque executa prompts de forma paralela e aleatória
- Porque é um modelo de linguagem especializado em NLP
- **Porque permite definir fluxos e interações entre múltiplos agentes**

---

**7. Qual é o objetivo principal ao criar agentes personalizados de IA?**

- Reduzir o consumo de memória do modelo base
- Aumentar a velocidade de resposta em consultas simples
- Substituir completamente os modelos de linguagem tradicionais
- **Permitir que o modelo atue de forma direcionada em contextos e tarefas específicas**

---

**8. Como o ReAct contribui para o raciocínio de um agente de IA?**

- Cria prompts automáticos para o modelo
- Apenas gera respostas baseadas em exemplos prévios
- Organiza os dados em memória de curto prazo
- **Integra o processo de pensar (reason) com o de agir (act), permitindo decisões mais eficazes**

---

**9. O que são 'observações' no contexto de um agente ReAct?**

- Os logs de erros do sistema
- As configurações de rede do ambiente de execução
- Os parâmetros de inicialização do modelo
- **Os resultados das ações executadas pelo agente, usados para ajustar o raciocínio**

---

**10. Por que agentes de IA precisam de memória?**

- Para armazenar códigos-fonte dos modelos
- Para exibir logs de depuração em tempo real
- Para reduzir o uso de CPU durante o processamento
- **Para manter o contexto e melhorar decisões baseadas em interações anteriores**

---

**11. Como a integração entre agentes e ferramentas externas amplia suas capacidades?**

- Apenas melhora a formatação das respostas geradas
- Limita o agente a respostas predefinidas para evitar erros
- Serve apenas para armazenar logs de execução
- **Permite que os agentes executem ações no mundo real, como acessar APIs e manipular dados**

---

**12. Como o LangGraph representa as interações entre agentes?**

- Como uma tabela de dados relacionais
- Como um script linear de execução
- Como uma lista de prompts fixos
- **Como um grafo de nós e fluxos de execução que define a comunicação entre agentes**

---

**13. O que diferencia um agente autônomo de um simples chatbot?**

- A limitação de operar apenas com textos curtos
- A presença de uma interface gráfica
- O uso exclusivo de mensagens pré-definidas
- **A capacidade de tomar decisões e executar ações baseadas em contexto**

---

**14. Qual o benefício da arquitetura multiagente em sistemas complexos?**

- Evitar o uso de protocolos como MCP
- Reduzir o número de modelos necessários
- Diminuir o tamanho do prompt
- **Dividir tarefas entre agentes especializados, aumentando eficiência e escalabilidade**

---

**15. Por que o MCP é considerado essencial para a interoperabilidade dos agentes?**

- Porque aumenta o poder de raciocínio do modelo
- Porque atua como servidor de cache de prompts
- Porque substitui as ferramentas (tools) do agente
- **Porque padroniza a forma como agentes e sistemas trocam informações**
